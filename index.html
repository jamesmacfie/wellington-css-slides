Things we do in JavaScript that can be done in CSS

Intro

  • Essentially, we can use CSS to keep track of state
  • :checked and :target
  • ~ general sibling selector
  • If you care about HTML structure then maybe some of this isn't for you

Disclaimer

  • Most of this is awesome
  • Some of it probably doesn't fit the bill for prod code

The humble checkbox

  • Two states: checked and unchecked
  • Used in conjunction with the general sibling selector ~
  • Input & label has to be written in a speciic way

The humble checkbox


  <input type="checkbox" id="someId" />
  <label for="someId">Toggle content</label>
  <div class="doSomething">
      <!-- Content -->
  </div>

The humble checkbox


  input { display: none; }

  .doSomething {
      /* State one */
  }

  input: checked ~ .doSomething {
      /* State two */
  }

Showing/hiding content

  • Elements: checkbox + content holder
  • Toggle content holder visibility on check

Showing/hiding content


  <input type="checkbox" id="showContent" />
  <!-- Content to always show -->
  <label for="showContent">Toggle content</label>
  <div class="showHideContent">
      <!-- Content to toggle -->
  </div>

Showing/hiding content


  .showHideContent {
      display: none;
  }

  .showToggle:checked ~.showHideContent {
      display: block;
  }

Off canvas menu

  • Elements: checkbox + sidebar holder + main content holder
  • Labels don't have to be directly beside their corresponding checkbox
  • Sidenote: you can also have more than one label per input if you need

Off canvas menu


  <input type="checkbox" class="menuToggle" id="menuToggle"/>
  <nav class="nav">
      <!-- Nav items -->
  </nav>
  <section>
      <header>
          <label for="menuToggle"></label>
      </header>
      <!-- Site content-->
  </section>

Off canvas menu


  .nav {
      transition: width 0.5s ease-in-out;
      width: 0;
  }

  .menuToggle:checked ~ .nav {
      width: 300px;
  }

Parrallex

  • Not sure if these are even cool or not anymore (were they ever cool?)
  • Properties: perspective, translateZ, scale

Parrallex


  <div class="parallax">
      <div class="parallaxGroup">
          <div class="parallaxLayer /* layerModifier */">
              <!-- Content -->
          </div>
      </div>
  </div>

Parrallex

  • Perspective
  • Also requires transform-style: preserve-3d to keep children in 3d space
  • TranslateZ - sets where in space each layer sits
  • Scale - to deal with different depth elements being smaller/larger

Modal

  • Simple visibility toggle
  • :target

Modal


  .modal {
      opacity: 0;
  }
  .modal:target {
      opacity: 1;
  }

Modal


  <div class="modal" id="modal">
      <div class="modalInner">
          <a class="modalX" href="#"></a>
          <!-- Content -->
      </div>
      <a href="#" class="modalClose"></a>
  </div>

Image gallery
